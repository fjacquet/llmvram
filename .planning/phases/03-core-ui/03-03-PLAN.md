  <name>Task 2: Fit Indicator and Recommendations</name>
  <files>

  <action>
    1. Create `src/components/outputs/FitIndicator.tsx`:
       - Props: `{ totalVRAM: Decimal, availableVRAM: number }` (Decimal from `decimal.js`, availableVRAM is `gpu.vram_gb` which is a plain number)
       - Calculate percentage: `const percentage = totalVRAM.div(availableVRAM).mul(100).toNumber()`
       - Three status tiers:
         - GREEN (<=80%): `{ label: 'Fits Comfortably', bgLight: 'bg-green-50', bgDark: 'dark:bg-green-900/20', border: 'border-green-200 dark:border-green-800', bar: 'bg-green-500', text: 'text-green-700 dark:text-green-400' }`
         - YELLOW (81-95%): `{ label: 'Tight Fit', bgLight: 'bg-yellow-50', bgDark: 'dark:bg-yellow-900/20', border: 'border-yellow-200 dark:border-yellow-800', bar: 'bg-yellow-500', text: 'text-yellow-700 dark:text-yellow-400' }`
         - RED (>95%): `{ label: 'Does Not Fit', bgLight: 'bg-red-50', bgDark: 'dark:bg-red-900/20', border: 'border-red-200 dark:border-red-800', bar: 'bg-red-500', text: 'text-red-700 dark:text-red-400' }`
       - Layout:
         ```
         +------------------------------------------+
         | [icon] Fits Comfortably          72%     |
         | Using 57.60 GB of 80 GB                  |
         | [============-------] progress bar       |
         +------------------------------------------+
         ```
       - Progress bar: outer `h-3 bg-gray-200 dark:bg-gray-700 rounded-full overflow-hidden`, inner div with percentage width (`style={{ width: \`${Math.min(percentage, 100)}%\` }}`) and status color
       - When >100%, cap the bar at 100% width but show actual percentage in text (e.g., "142%")
       - Use status icons (not emoji): checkmark SVG for green, warning triangle SVG for yellow, X-circle SVG for red. Use inline SVG with `currentColor`.
       - `role="status"` and `aria-live="polite"` so screen readers announce changes when inputs update

    2. Create `src/components/outputs/Recommendations.tsx`:
       - Props: `{ model: Model, gpu: GPU, breakdown: InferenceVRAMBreakdown, currentQuantization: QuantizationFormat, currentSequenceLength: number }`
       - Import types from `@utils/schemas` (Model, GPU) and `@engines/types` (InferenceVRAMBreakdown, QuantizationFormat)
       - Calculate deficit: `const deficit = breakdown.total.minus(gpu.vram_gb).toNumber()`
       - Generate recommendations as an array of `{ title: string, description: string, impact: string }`:

         a. **Lower quantization** (only if current quantization is NOT already the smallest):
            - If currently fp32/fp16/bf16: suggest GPTQ or AWQ 4-bit
            - If currently int8: suggest int4/nf4/GPTQ
            - If currently 4-bit variants: suggest GGUF Q4_0 (most compressed)
            - Estimate savings: "Could save ~X GB" based on weight ratio (e.g., fp16->gptq saves ~50% of model weights)
            - Skip this recommendation if already at gguf-q4_0

         b. **Reduce context length** (only if current > 512):
            - Suggest halving: "Reduce from {current} to {current/2} tokens"
            - Estimate savings: "Could save ~X GB" = approximately half the KV cache
            - Skip if already at 512

         c. **Use KV cache quantization** (only if current is fp16):
            - Suggest int4 KV cache: "Reduce KV cache precision to INT4"
            - Estimate savings: ~75% of KV cache

         d. **Use multiple GPUs** (always show):
            - Calculate: `Math.ceil(breakdown.total.toNumber() / gpu.vram_gb)` GPUs needed
            - "Use {N}x {gpu.name} with tensor parallelism (available in Phase 4)"

         e. **Upgrade GPU** (show if a larger common GPU exists):
            - If VRAM < 24: suggest "RTX 4090 (24 GB)" or "RTX 5090 (32 GB)"
            - If VRAM < 48: suggest "A6000 (48 GB)"
            - If VRAM < 80: suggest "A100/H100 (80 GB)"
            - If VRAM >= 80: suggest "Multiple GPUs" only

       - Render as a card with blue border:
         ```
         +------------------------------------------+
         | Recommendations                           |
         | Need X.XX GB more VRAM. Try:             |
         |                                          |
         | 1. Lower quantization                     |
         |    Try GPTQ 4-bit — saves ~XX GB         |
         |                                          |
         | 2. Reduce context length                  |
         |    From 4096 to 2048 — saves ~XX GB      |
         +------------------------------------------+
         ```
       - Each recommendation: numbered, bold title, description with estimated savings
       - Style: `rounded-lg border-2 border-blue-200 dark:border-blue-800 bg-blue-50 dark:bg-blue-900/20 p-6`
       - `role="complementary"` and `aria-label="Optimization recommendations"`

    Important: Both components receive all data as props. They do NOT access the Zustand store directly. This keeps them pure display components that are easy to test and reuse.
  </action>

    - Recommendations generates at least 2 suggestions when given a config that doesn't fit
