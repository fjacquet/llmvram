  <action>
    1. Install @headlessui/react (needed by Plan 02 for accessible combobox selectors):
       `npm install @headlessui/react`

    2. Create `src/index.css` with Tailwind v4 CSS-first configuration:
       ```css
       @import "tailwindcss";

       @custom-variant dark (&:where(.dark, .dark *));

       @theme {
         --color-surface: #ffffff;
         --color-surface-dark: #0f172a;
         --color-panel: #f8fafc;
         --color-panel-dark: #1e293b;
       }
       ```
       The `@custom-variant dark` line is CRITICAL for Tailwind v4 class-based dark mode. Without it, `dark:` utility classes silently fail (Pitfall 1 from research). Do NOT use `@media (prefers-color-scheme: dark)` — we need class-based toggling.

    3. Update `src/main.tsx` to import the CSS file. Add `import './index.css'` before the React imports. Keep all existing code (StrictMode, createRoot).

    4. Add FOUC prevention inline script to `index.html` inside `<head>`, BEFORE any stylesheets or the module script:
       ```html
       <script>
         (function() {
           var stored = localStorage.getItem('llmvram-ui-preferences')
           if (stored) {
             try {
               var parsed = JSON.parse(stored)
               if (parsed.state && parsed.state.isDarkMode) {
                 document.documentElement.classList.add('dark')
               }
             } catch(e) {}
           }
         })()
       </script>
       ```
       This synchronous script runs before first paint, preventing the white flash when dark mode is enabled. Uses `var` not `const`/`let` for maximum browser compatibility. The key name `llmvram-ui-preferences` must match the Zustand persist store name exactly.
  </action>
  <verify>

    - `@headlessui/react` appears in package.json dependencies
    - `index.html` contains the FOUC prevention script
  </verify>
