  <action>
    1. Create `src/components/inputs/ModelSelector.tsx`:
       - Use Headless UI `Combobox`, `ComboboxInput`, `ComboboxButton`, `ComboboxOptions`, `ComboboxOption` from `@headlessui/react`
       - Import models data: `import modelsData from '@data/models.json'` and validate with `validateModels` from `@utils/schemas`
       - Import `createCustomModel` and `CustomModelInput` from `@types/model`
       - Connect to store: `const { selectedModel, setSelectedModel } = useUIStore()`
       - Implement search filtering: filter models by name (case-insensitive `.toLowerCase().includes()`)
       - Display each model option as: `{name} ({num_parameters_billion}B)` with architecture badge for MoE models
       - Add a "Custom model..." option at the bottom of the dropdown, separated by a border
       - When "Custom model..." is selected, show an inline form below the combobox with fields:
         - Name (text input, required)
         - Parameter count in billions (number input, required, min 0.1)
         - Hidden size (number input, optional, placeholder "4096")
         - Number of layers (number input, optional, placeholder "32")
         - Number of attention heads (number input, optional, placeholder "32")
       - Form has "Add Model" button that calls `createCustomModel(input)` and then `setSelectedModel(model)`
       - Style with Tailwind: rounded borders, dark mode variants (dark:bg-gray-800, dark:border-gray-600, dark:text-white), focus ring
       - ComboboxOptions: `max-h-60 overflow-auto rounded-lg shadow-lg` with proper z-index (`z-10`) for overlay
       - Add `aria-label="Select model"` on the combobox wrapper

    2. Create `src/components/inputs/GPUSelector.tsx`:
       - Same pattern as ModelSelector but for GPUs
       - Import gpus data: `import gpusData from '@data/gpus.json'` and validate with `validateGPUs`
       - Import `createCustomGPU` and `CustomGPUInput` from `@types/gpu`
       - Connect to store: `const { selectedGPU, setSelectedGPU } = useUIStore()`
       - Display each GPU option as: `{name} ({vram_gb} GB)` with tier badge (datacenter/consumer/apple-silicon)
       - Group GPUs by manufacturer in the dropdown (NVIDIA header, AMD header, Apple header) using non-selectable divider options
       - Custom GPU inline form fields:
         - Name (text input, required)
         - VRAM in GB (number input, required, min 1)
         - Memory bandwidth in GB/s (number input, optional, placeholder "900")
         - FP16 TFLOPS (number input, optional, placeholder "312")
       - Form has "Add GPU" button that calls `createCustomGPU(input)` and then `setSelectedGPU(gpu)`
       - Same Tailwind styling as ModelSelector for consistency
       - Add `aria-label="Select GPU"` on the combobox wrapper

    Important: Do NOT use native `<select>` elements. Headless UI Combobox provides keyboard navigation, screen reader support, and search filtering that native select cannot.

    Important: Use `useState` for the custom form's local state (name, params, etc.). Do NOT put temporary form state in the Zustand store. Only call `setSelectedModel`/`setSelectedGPU` when the user submits the custom form.
  </action>
  <verify>
    - `npm run typecheck` passes (Headless UI types resolve, model/GPU types correct)
    - `npm run lint` passes
    - Both components render without errors when imported into a test page
    - ModelSelector shows all models from models.json in dropdown
    - GPUSelector shows all GPUs from gpus.json in dropdown
  </verify>
  <done>
    ModelSelector provides searchable combobox with curated models + custom model form. GPUSelector provides searchable combobox with curated GPUs grouped by manufacturer + custom GPU form. Both update Zustand store on selection.
  </done>
