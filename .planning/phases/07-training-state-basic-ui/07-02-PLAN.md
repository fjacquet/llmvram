    Container component that assembles all training-specific inputs.

    ```typescript

      return (
        <div className="space-y-4">

          </h3>
          <TrainingMethodPicker />
          <OptimizerPicker />
          <PrecisionPicker />
        </div>
      )
    }
    ```

    This is intentionally simple — it's a layout container. LoRA-specific inputs (rank, alpha, target modules percentage) are NOT included in Phase 7; they will be added when the training calculation is wired in a later phase (the store already has the state ready).

    **2. Modify `src/components/layout/InputPanel.tsx`:**

    Add the ModeToggle at the TOP of the input panel (before Model Configuration), and conditionally render TrainingPanel when in training mode.

    Changes:
    - Import `ModeToggle` from `@components/inputs/ModeToggle`
    - Import `TrainingPanel` from `@components/inputs/TrainingPanel`
    - Read `mode` from store: `const mode = useUIStore((s) => s.mode)`
    - Add ModeToggle as the first element inside the `space-y-6` container, before the "Model Configuration" section
    - Add an `<hr>` separator after ModeToggle
    - When `mode === 'training'`, show the TrainingPanel section after the GPU Selection section (before Parameters). Add an `<hr>` before TrainingPanel.
    - The existing inference-specific sections (Parameters with SequenceLength, BatchSize, KVQuantization) should remain visible in BOTH modes — these parameters are relevant for training too (batch size, sequence length affect training VRAM)
    - The Hardware Configuration and Offloading sections remain visible in BOTH modes (GPU selection is needed for training too)

    Layout structure after modification:
    ```
    [ModeToggle]
    ---
    [Model Configuration: ModelSelector, QuantizationPicker]
    ---
    [GPU Selection: GPUSelector]
    ---
    [Hardware Configuration (if GPU selected)]
    ---
    [Offloading (if GPU selected)]
    ---
    [Training Configuration (if mode=training): TrainingMethodPicker, OptimizerPicker, PrecisionPicker]
    ---
    [Parameters: SequenceLength, BatchSize, KVQuantization]
    ```

    The key principle: ModeToggle is always visible at the top. TrainingPanel only appears when `mode === 'training'`. All other sections remain visible in both modes.
