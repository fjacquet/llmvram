  <behavior>
    Three optimization functions with clear I/O contracts:

    1. calculateEffectiveBatchSize(perDeviceBatch, accumulationSteps, numGPUs):
       - (2, 4, 1) -> 8
       - (2, 4, 4) -> 32
       - (1, 8, 1) -> 8
       - (4, 1, 1) -> 4

    2. applyGradientCheckpointing(activationMemoryGB: Decimal, enabled: boolean):
       - (Decimal(10.0), true) -> Decimal(4.0) (60% reduction, retain 40%)
       - (Decimal(10.0), false) -> Decimal(10.0) (identity)
       - (Decimal(0), true) -> Decimal(0) (zero stays zero)

    3. applyFlashAttention(activationMemoryGB: Decimal, sequenceLength: number, enabled: boolean):
       - (Decimal(10.0), 1024, true) -> Decimal(8.5) (15% reduction, retain 85%)
       - (Decimal(10.0), 4096, true) -> Decimal(5.0) (50% reduction, retain 50%)
       - (Decimal(10.0), 16384, true) -> Decimal(3.0) (70% reduction, retain 30%)
       - (Decimal(10.0), 4096, false) -> Decimal(10.0) (identity when disabled)

    4. Combined (multiplicative stacking):
       - Base 10 GB + checkpointing + flash (4096 seq):
         10 * 0.4 * 0.5 = 2.0 GB

    5. Updated training engines accept optional optimization params:
       - calculateFullFineTuningVRAM({...existing, gradientCheckpointing: true, flashAttention: true})
         -> activations field reflects optimized (reduced) value, total is lower
       - calculateLoRAFineTuningVRAM({...existing, gradientCheckpointing: true})
         -> activations reduced by checkpointing
       - calculateQLoRAFineTuningVRAM({...existing, flashAttention: true})
         -> activations reduced by Flash Attention
       - All functions work unchanged when optimization params omitted (backward compat)
  </behavior>
