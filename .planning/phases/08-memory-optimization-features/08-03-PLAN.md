  <name>Task 1: Create useTrainingCalculation hook</name>
  <files>src/hooks/useTrainingCalculation.ts</files>
  <action>
    Create a new hook that reads training configuration from the Zustand store and calls
    the appropriate training engine function based on the selected fine-tuning method.

    Pattern: Follow useInferenceCalculation.ts structure but use useMemo (synchronous)
    instead of useEffect + Worker (training calculations are fast, no need for async).

    1. Define return type interface:
       ```typescript
       export interface UseTrainingCalculationResult {
         result: TrainingVRAMBreakdown | LoRAVRAMBreakdown | null
         error: string | null
       }
       ```

    2. Export function useTrainingCalculation():
       - Read from useUIStore: selectedModel, trainingMethod, optimizer, trainingPrecision,
         batchSize, sequenceLength, loraRank, targetModulesPercent,
         gradientCheckpointing, flashAttention
       - Early return { result: null, error: null } if selectedModel is null
       - Use useMemo with all parameters as dependencies
       - Inside useMemo, use try/catch:
         a. Switch on trainingMethod:
            - 'full': call calculateFullFineTuningVRAM({
                model: selectedModel, trainingPrecision, optimizer, batchSize, sequenceLength,
                gradientCheckpointing, flashAttention, sequenceLengthForFlash: sequenceLength
              })
            - 'lora': call calculateLoRAFineTuningVRAM({
                model: selectedModel, trainingPrecision, optimizer, batchSize, sequenceLength,
                loraRank, targetModulesPercent,
                gradientCheckpointing, flashAttention, sequenceLengthForFlash: sequenceLength
              })
            - 'qlora': call calculateQLoRAFineTuningVRAM({
                model: selectedModel, optimizer, batchSize, sequenceLength,
                loraRank, targetModulesPercent,
                gradientCheckpointing, flashAttention, sequenceLengthForFlash: sequenceLength
              })
         b. Return { result: breakdown, error: null }
         c. Catch errors: return { result: null, error: err.message }

    NOTE: The exact parameter names for optimization params depend on how 08-01 added them
    to the engine functions. Check the 08-01 SUMMARY for the actual parameter names and
    adapt accordingly. The key is that gradientCheckpointing (boolean) and flashAttention
    (boolean) from the store get passed through to the engine functions, along with
    sequenceLength for Flash Attention's sequence-dependent reduction factor.

    3. Import types from '@engines/types' (TrainingVRAMBreakdown, LoRAVRAMBreakdown)
    4. Import engine functions from '@engines/training' and '@engines/lora'
    5. Import { useMemo } from 'react'
    6. Import { useUIStore } from '@store/uiStore'

    Add JSDoc with usage example similar to useInferenceCalculation.
  </action>

    - Hook calls correct engine function based on trainingMethod
