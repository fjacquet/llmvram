**Function 1: calculateLoRAAdapterParams**
- Input: `{ hiddenSize: number, numHiddenLayers: number, rank: number, targetModulesPercent: number }`
- Output: Decimal (raw parameter count, NOT billions)
- Formula:
  1. `targetModuleCount = Math.max(1, Math.round(TOTAL_TARGETABLE_MODULES_PER_LAYER * targetModulesPercent / 100))`
  2. Each LoRA adapter adds two low-rank matrices per target module: A(d, r) + B(r, d) = 2 * rank * d params
  3. `paramsPerLayer = 2 * rank * hiddenSize * targetModuleCount`
  4. `totalAdapterParams = paramsPerLayer * numHiddenLayers`
- NOTE: alpha does NOT affect parameter count (only scales output: alpha/rank * B @ A @ x)

Test cases (use 7B model: hidden=4096, layers=32):
- rank=16, targetModulesPercent=30 (→ 2 modules):
  → 2 * 16 * 4096 * 2 * 32 = 8,388,608 (~8.4M, 0.12% of 7B)
- rank=16, targetModulesPercent=50 (→ 4 modules):
  → 2 * 16 * 4096 * 4 * 32 = 16,777,216 (~16.8M, 0.24% of 7B)
- rank=16, targetModulesPercent=100 (→ 7 modules):
  → 2 * 16 * 4096 * 7 * 32 = 29,360,128 (~29.4M, 0.42% of 7B)
- rank=64, targetModulesPercent=50 (→ 4 modules):
  → 2 * 64 * 4096 * 4 * 32 = 67,108,864 (~67.1M, 0.96% of 7B)
- rank=256, targetModulesPercent=100 (→ 7 modules, maximum):
  → 2 * 256 * 4096 * 7 * 32 = 469,762,048 (~470M, 6.7% of 7B)

**Function 2: calculateLoRAFineTuningVRAM**
- Input: `{ model: Model, trainingPrecision: TrainingPrecision, optimizer: OptimizerType, batchSize: number, sequenceLength: number, loraRank: number, targetModulesPercent: number }`
- Output: LoRAVRAMBreakdown
- Components:
  1. `baseWeights` = calculateModelWeightVRAM(model.num_parameters_billion, trainingPrecision === 'fp32' ? 'fp32' : trainingPrecision === 'fp16' ? 'fp16' : 'bf16') — frozen base at training precision
  2. `adapterParameters` = calculateLoRAAdapterParams(...)
  3. `adapterParamsBillion` = adapterParameters / 1e9
  4. `adapterWeights` = adapterParamsBillion * 1e9 * 2 / BYTES_PER_GB — adapters always FP16/BF16
  5. `masterWeights` = if mixed precision: adapterParamsBillion * 1e9 * 4 / BYTES_PER_GB (FP32 master copy of adapters only)
  6. `gradients` = adapterParamsBillion * 1e9 * GRADIENT_BYTES[precision] / BYTES_PER_GB — gradients ONLY for adapters
  7. `optimizerStates` = calculateOptimizerStateMemory(adapterParamsBillion, optimizer) — ONLY for adapter params (PITFALLS.md #3)
  8. `activations` = calculateTrainingActivationMemory(model, batchSize, sequenceLength)
  9. `frameworkOverhead` = TRAINING_FRAMEWORK_OVERHEAD_GB
  10. `modelWeights` = baseWeights (for interface compatibility — this is the frozen base)
  11. `total` = baseWeights + adapterWeights + masterWeights + gradients + optimizerStates + activations + frameworkOverhead
  12. `trainableParameters` = adapterParamsBillion
  13. `totalParameters` = model.num_parameters_billion
  14. `method` = 'lora'

CRITICAL: Optimizer states and gradients apply ONLY to adapter parameters, NOT frozen base (PITFALLS.md #3).

Test case: 7B model, BF16, AdamW, batch=1, seq=2048, rank=16, targetModules=50%:
- baseWeights: ~13.04 GB (frozen BF16)
- adapterParams: ~16.8M
- adapterWeights: ~0.031 GB (FP16)
- masterWeights: ~0.063 GB (FP32 copy of adapters)
- gradients: ~0.031 GB (only adapters)
- optimizerStates: ~0.125 GB (only adapters, AdamW 8 bytes)
- activations: ~5.0 GB
- overhead: 1.5 GB
- total: ~19.8 GB (fits on 24GB GPU!)

**Function 3: calculateQLoRAFineTuningVRAM**
- Input: `{ model: Model, optimizer: OptimizerType, batchSize: number, sequenceLength: number, loraRank: number, targetModulesPercent: number }`
- Output: LoRAVRAMBreakdown
- QLoRA is a THREE-PRECISION architecture (PITFALLS.md #4):
  1. `baseWeights` = calculateModelWeightVRAM(model.num_parameters_billion, 'nf4') — frozen base in 4-bit NF4
  2. `adapterParameters` = calculateLoRAAdapterParams(...)
  3. `adapterParamsBillion` = adapterParameters / 1e9
  4. `adapterWeights` = adapterParamsBillion * 1e9 * 2 / BYTES_PER_GB — adapters ALWAYS FP16
  5. `masterWeights` = adapterParamsBillion * 1e9 * 4 / BYTES_PER_GB — FP32 master of adapters (always mixed precision)
  6. `gradients` = adapterParamsBillion * 1e9 * 2 / BYTES_PER_GB — FP16 gradients for adapters only
  7. `optimizerStates` = calculateOptimizerStateMemory(adapterParamsBillion, optimizer) — FP32 for adapters only
  8. `activations` = calculateTrainingActivationMemory(model, batchSize, sequenceLength)
  9. `frameworkOverhead` = TRAINING_FRAMEWORK_OVERHEAD_GB
  10. `modelWeights` = baseWeights
  11. `total` = sum of all
  12. `trainableParameters` = adapterParamsBillion
  13. `totalParameters` = model.num_parameters_billion
  14. `method` = 'qlora'

NOTE: QLoRA does NOT accept trainingPrecision — base is always NF4, adapters always FP16, states always FP32.

Test case: 7B model, AdamW, batch=1, seq=2048, rank=16, targetModules=50%:
- baseWeights: ~3.26 GB (NF4: 7B * 0.5 / 1024^3)
- adapterWeights: ~0.031 GB
- masterWeights: ~0.063 GB
- gradients: ~0.031 GB
- optimizerStates: ~0.125 GB
- activations: ~5.0 GB
- overhead: 1.5 GB
- total: ~10.0 GB (fits on 12GB GPU with small batch!)

Additional test: QLoRA 70B model should still require significant VRAM for base weights:
- baseWeights: ~32.60 GB (NF4: 70B * 0.5)
- total: ~40+ GB
