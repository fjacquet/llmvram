<feature>
  <name>Core Training VRAM Engine (Full Fine-Tuning)</name>
  <files>src/engines/training.ts, src/engines/training.test.ts</files>
  <behavior>
**Function 1: calculateOptimizerStateMemory**
- Input: `trainableParamsBillion: number, optimizer: OptimizerType`
- Output: Decimal (GB)
- Formula: `trainableParams * OPTIMIZER_STATE_BYTES[optimizer] / BYTES_PER_GB`
- All optimizer states are FP32 regardless of training precision (PITFALLS.md #2)

Test cases:
- `(7.0, 'adamw')` → 7e9 * 8 / 1024^3 = ~52.15 GB
- `(7.0, 'sgd-momentum')` → 7e9 * 4 / 1024^3 = ~26.08 GB
- `(7.0, 'adamw-8bit')` → 7e9 * 2 / 1024^3 = ~13.04 GB
- `(7.0, 'adafactor')` → 7e9 * 4 / 1024^3 = ~26.08 GB
- `(0.5, 'adamw')` → 0.5e9 * 8 / 1024^3 = ~3.73 GB (small model)

**Function 2: calculateTrainingActivationMemory**
- Input: `model: Model, batchSize: number, sequenceLength: number`
- Output: Decimal (GB)
- Formula: `batchSize * sequenceLength * hiddenSize * numLayers * 10 * 2 / BYTES_PER_GB`
  - `10` = approximate multiplier per layer (attention QKV + MLP up/gate/down + residuals)
  - `2` = bytes per element (FP16/BF16 activations)
- For MoE models: use active param ratio to scale (same logic as inference.ts)

Test cases:
- 7B dense model (hidden=4096, layers=32), batch=1, seq=2048:
  → 1 * 2048 * 4096 * 32 * 10 * 2 / 1024^3 = ~5.0 GB
- Same model, batch=4:
  → ~20.0 GB (linear with batch)
- Same model, seq=4096:
  → ~10.0 GB (linear with sequence)

**Function 3: calculateFullFineTuningVRAM**
- Input: `{ model: Model, trainingPrecision: TrainingPrecision, optimizer: OptimizerType, batchSize: number, sequenceLength: number }`
- Output: TrainingVRAMBreakdown
- Components:
  1. `modelWeights` = paramsBillion * 1e9 * WEIGHT_BYTES[precision] / BYTES_PER_GB
  2. `masterWeights` = if mixed precision (fp16/bf16): paramsBillion * 1e9 * 4 / BYTES_PER_GB, else 0
  3. `gradients` = paramsBillion * 1e9 * GRADIENT_BYTES[precision] / BYTES_PER_GB
  4. `optimizerStates` = calculateOptimizerStateMemory(paramsBillion, optimizer)
  5. `activations` = calculateTrainingActivationMemory(model, batchSize, sequenceLength)
  6. `frameworkOverhead` = TRAINING_FRAMEWORK_OVERHEAD_GB
  7. `total` = sum of all above
  8. `trainableParameters` = new Decimal(paramsBillion)
  9. `totalParameters` = new Decimal(paramsBillion)
  10. `method` = 'full'

Test cases:
- 7B model, BF16, AdamW, batch=1, seq=2048:
  - modelWeights: ~13.04 GB (7B * 2 / 1024^3)
  - masterWeights: ~26.08 GB (7B * 4 / 1024^3, mixed precision)
  - gradients: ~13.04 GB (same as modelWeights in BF16)
  - optimizerStates: ~52.15 GB (AdamW 8 bytes/param FP32)
  - activations: ~5.0 GB
  - overhead: 1.5 GB
  - total: ~110.81 GB
- 7B model, FP32, AdamW, batch=1, seq=2048:
  - modelWeights: ~26.08 GB (7B * 4)
  - masterWeights: 0 GB (no master copy in pure FP32)
  - gradients: ~26.08 GB
  - optimizerStates: ~52.15 GB
  - total: higher than BF16 (no master weights, but heavier base)
- 70B model, BF16, AdamW, batch=1, seq=2048:
  - total: much higher (10x scale from 7B)
