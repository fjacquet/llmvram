<objective>
Extend the Zustand store, Web Worker, and calculation hook to support multi-GPU parameters and results, connecting the new engine (Plan 01) to the existing data flow pipeline.

Purpose: The store-worker-hook pipeline is the plumbing that connects user inputs to calculation results. Without this, the engine from Plan 01 has no way to receive parameters from the UI or deliver results to the output components. This plan is a clean vertical integration that maintains the existing architecture patterns (Decimal serialization, Worker fallback, persist middleware).

Output: Modified store with multi-GPU state, Worker handling multi-GPU calculations, hook returning multi-GPU results.
</objective>

         return {
           numGPUs: serialized.numGPUs,
           strategy: serialized.strategy as ShardingStrategy,
           perGPU: {
             modelWeights: new Decimal(serialized.perGPU.modelWeights),
             kvCache: new Decimal(serialized.perGPU.kvCache),
             activations: new Decimal(serialized.perGPU.activations),
             frameworkOverhead: new Decimal(serialized.perGPU.frameworkOverhead),
             communicationOverhead: new Decimal(serialized.perGPU.communicationOverhead),

           totalPerGPU: new Decimal(serialized.totalPerGPU),

       }
