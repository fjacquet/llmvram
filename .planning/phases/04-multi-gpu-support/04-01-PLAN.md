  <behavior>
    The multi-GPU engine takes a single-GPU InferenceVRAMBreakdown (from existing calculateInferenceVRAM), a Model, a GPU, numGPUs (1-8), and a ShardingStrategy, and returns a MultiGPUVRAMBreakdown with per-GPU memory distribution.

    ## Types to add in src/engines/types.ts

    ```typescript
    export type ShardingStrategy = 'tensor-parallel' | 'pipeline-parallel'

    export type InterconnectType = 'nvlink-4' | 'nvlink-5' | 'pcie-4' | 'pcie-5' | 'none'

    export interface InterconnectSpec {
      type: InterconnectType
      bandwidthGBps: number
      recommendedMaxTPDegree: number
    }

    export interface MultiGPUVRAMBreakdown {
      numGPUs: number
      strategy: ShardingStrategy
      perGPU: {
        modelWeights: Decimal
        kvCache: Decimal
        activations: Decimal
        frameworkOverhead: Decimal
        communicationOverhead: Decimal
        total: Decimal
      }
      replicatedMemory: Decimal
      totalPerGPU: Decimal
      utilizationPercent: Decimal
      singleGPUBaseline: Decimal
    }

    export interface InterconnectValidation {
      valid: boolean
      warning: string | null
      interconnect: InterconnectSpec
    }
    ```

    ## Constants to add in src/engines/constants.ts

    ```typescript
    export const INTERCONNECT_SPECS: Record<InterconnectType, InterconnectSpec>
    // nvlink-4: 900 GB/s, maxTP=8
    // nvlink-5: 1800 GB/s, maxTP=8
    // pcie-4: 64 GB/s, maxTP=2
    // pcie-5: 128 GB/s, maxTP=4
    // none: 0 GB/s, maxTP=1

    export const NCCL_BUFFER_PER_PEER_GB = new Decimal(0.2)  // 200MB per peer
    export const EMBEDDING_WEIGHT_FRACTION = new Decimal(0.03)  // ~3% of model weights
    export const TP_COMMUNICATION_OVERHEAD = new Decimal(0.12)  // 12% for tensor parallel
    export const PP_COMMUNICATION_OVERHEAD = new Decimal(0.05)  // 5% for pipeline parallel
    export const PP_ACTIVATION_STASHING_OVERHEAD = new Decimal(0.12)  // 12% activation stashing
    export const MOE_MULTI_GPU_OVERHEAD = new Decimal(0.15)  // 15% extra for MoE expert routing
    ```

    ## Test cases (input -> expected output)

    **Tensor Parallelism - Llama 3 70B GPTQ on 4x H100:**
    - singleGPU.modelWeights = ~39.12 GB, singleGPU.total = ~41.83 GB
    - layerNormMemory = 80 * 2 * 8192 * 4 / 1024^3 = ~0.0049 GB
    - embeddingEstimate = 39.12 * 0.03 = ~1.17 GB
    - replicatedMemory = ~1.18 GB
    - shardableWeights = 39.12 - 1.18 = ~37.94 GB
    - weightsPerGPU = 37.94 / 4 + 1.18 = ~10.66 GB
    - kvCachePerGPU = singleGPU.kvCache / 4
    - activationsPerGPU = singleGPU.activations / 4
    - ncclBuffers = 0.2 * 3 = 0.6 GB
    - frameworkOverheadPerGPU = 1.0 + 0.6 = 1.6 GB
    - communicationOverhead = 10.66 * 0.12 = ~1.28 GB
    - totalPerGPU = weightsPerGPU + kvCache/4 + activations/4 + 1.6 + 1.28

    **Pipeline Parallelism - Llama 3 70B GPTQ on 4 GPUs:**
    - weightsPerGPU = 39.12 / 4 = ~9.78 GB
    - kvCachePerGPU = singleGPU.kvCache (NOT divided!)
    - baseActivationsPerGPU = singleGPU.activations / 4
    - activationsPerGPU = baseActivationsPerGPU * 1.12 (stashing overhead)
    - frameworkOverheadPerGPU = 1.0 GB (no NCCL buffers for PP)
    - communicationOverhead = 9.78 * 0.05 = ~0.49 GB

    **MoE model (Mixtral 8x7B) adds 15% communication overhead:**
    - communicationOverhead for TP = weightsPerGPU * 0.12 * 1.15

    **Single GPU passthrough:**
    - numGPUs=1 returns: replicatedMemory=0, communicationOverhead=0, totalPerGPU=singleGPU.total

    **Interconnect validation:**
    - nvlink-4 GPU, TP=4 -> valid=true, warning=null
    - pcie-4 GPU, TP=4 -> valid=true, warning="PCIe 4.0 may have significant communication overhead..."
    - none interconnect, numGPUs=2 -> valid=false, warning="does not support multi-GPU..."

    **resolveInterconnect mapping:**
    - GPU with interconnect='nvlink-4' -> InterconnectType 'nvlink-4'
    - GPU with interconnect='nvlink' (generic) -> 'nvlink-4'
    - GPU with interconnect='none' or undefined, tier='datacenter' -> 'pcie-5'
    - GPU with interconnect='none' or undefined, tier='consumer' -> 'pcie-4'
    - GPU with interconnect='unified' (Apple) -> 'none'
    - GPU with interconnect='infinity-fabric' -> 'pcie-5' (AMD MI300X uses PCIe-like bandwidth)

    **Edge cases:**
    - numGPUs < 1 or > 8 throws error
    - numGPUs=1 always valid regardless of interconnect
  </behavior>

       - frameworkOverheadPerGPU = singleGPU.frameworkOverhead (no NCCL)
       - moeMultiplier = isMoE ? (1 + MOE_MULTI_GPU_OVERHEAD) : 1
       - communicationOverhead = weightsPerGPU * PP_COMMUNICATION_OVERHEAD * moeMultiplier
       - totalPerGPU = sum of all components
