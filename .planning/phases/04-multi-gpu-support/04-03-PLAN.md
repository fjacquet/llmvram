    ## Engine (src/engines/offloading.ts):

    Create a pure calculation function for offloaded VRAM:

    ```ts
    import Decimal from 'decimal.js'
    import type { InferenceVRAMBreakdown, OffloadingConfig, OffloadedVRAMBreakdown } from './types'

    /**
     * Calculate VRAM breakdown after offloading model weights and/or KV cache.
     *
     * When offloading by percentage: offloadPercentage% of model weights move off GPU.
     * When offloading by layers: (offloadLayers / totalLayers) fraction of weights move off GPU.
     * KV cache offloading moves entire KV cache to CPU/RAM (independent toggle).
     *
     * Performance impact estimates:
     * - CPU/RAM offloading: PCIe bandwidth limited, ~2-5x slower for small offloads, ~15-50x for large
     * - NVMe offloading: Even slower, ~10-100x depending on SSD speed and amount
     * - KV cache offload adds latency per token (every decode step reads from CPU)
     */
    export function calculateOffloadedVRAM(
      breakdown: InferenceVRAMBreakdown,
      config: OffloadingConfig,
      totalLayers?: number,
    ): OffloadedVRAMBreakdown
    ```

    Implementation logic:
    1. If `!config.enabled`, return breakdown unchanged with zero offloaded and no impact.
    2. Calculate effective offload fraction:
       - If mode === 'percentage': `fraction = config.offloadPercentage / 100`
       - If mode === 'layers': `fraction = Math.min(config.offloadLayers / (totalLayers ?? 1), 1.0)`
       - Clamp fraction to [0, 1]
    3. offloaded.modelWeights = breakdown.modelWeights * fraction
    4. onDevice.modelWeights = breakdown.modelWeights * (1 - fraction)
    5. If kvCacheOffload:
       - offloaded.kvCache = breakdown.kvCache (full KV cache offloaded)
       - onDevice.kvCache = Decimal(0)
    6. Else:
       - offloaded.kvCache = Decimal(0)
       - onDevice.kvCache = breakdown.kvCache
    7. onDevice.activations = breakdown.activations (activations always on GPU)
    8. onDevice.frameworkOverhead = breakdown.frameworkOverhead (always on GPU)
    9. onDevice.total = sum of all onDevice components
    10. offloaded.total = offloaded.modelWeights + offloaded.kvCache
    11. Estimate slowdownFactor and performanceImpact string:
        - offloadedGB = offloaded.total.toNumber()
        - For CPU/RAM target:
          - offloadedGB <= 4: slowdown 2, "~2x slower"
          - offloadedGB <= 8: slowdown 3, "~2-5x slower"
          - offloadedGB <= 24: slowdown 8, "~5-15x slower"
          - offloadedGB > 24: slowdown 25, "~15-50x slower"
        - For NVMe target:
          - Multiply CPU/RAM slowdown by ~3x (NVMe is much slower than RAM)
          - offloadedGB <= 8: slowdown 10, "~5-15x slower"
          - offloadedGB > 8: slowdown 50, "~30-100x slower"
        - If kvCacheOffload: append " (KV cache on CPU adds per-token latency)"
    12. Return { onDevice, offloaded, performanceImpact, slowdownFactor }
