  <files>src/store/uiStore.ts</files>
  <action>
    Add the following fields to the UIState interface:

    1. `frameworkPreset: FrameworkPreset` -- defaults to 'none'
    2. `cpuOffloadOptimizer: boolean` -- defaults to false

    Add the following actions to UIState interface:

    3. `setFrameworkPreset: (preset: FrameworkPreset) => void` -- with auto-apply logic (see below)
    4. `setCpuOffloadOptimizer: (enabled: boolean) => void`

    Import FrameworkPreset type and FRAMEWORK_PRESETS constant from '@engines/frameworks'.

    Implement setFrameworkPreset with auto-apply logic in the store:
    ```
    setFrameworkPreset: (preset) => set((state) => {
      const config = FRAMEWORK_PRESETS[preset]
      const updates: Partial<UIState> = { frameworkPreset: preset }

      // Mode enforcement: vLLM/TGI are inference-only
      if (config.mode === 'inference') {
        updates.mode = 'inference'
      }

      // Auto-apply optimizations from preset config
      if (config.autoOptimizations.gradientCheckpointing !== undefined) {
        updates.gradientCheckpointing = config.autoOptimizations.gradientCheckpointing
      }
      if (config.autoOptimizations.flashAttention !== undefined) {
        updates.flashAttention = config.autoOptimizations.flashAttention
      }
      if (config.autoOptimizations.optimizer !== undefined) {
        updates.optimizer = config.autoOptimizations.optimizer
      }

      // Reset CPU offload when switching away from DeepSpeed
      if (!preset.startsWith('deepspeed-')) {
        updates.cpuOffloadOptimizer = false
      }

      return updates
    })
    ```

    Implement setCpuOffloadOptimizer as simple setter:
    ```
    setCpuOffloadOptimizer: (enabled) => set({ cpuOffloadOptimizer: enabled })
    ```

    IMPORTANT: Do NOT add a separate zeroStage field. The ZeRO stage is derived from
    frameworkPreset: deepspeed-zero1 -> 'zero-1', deepspeed-zero2 -> 'zero-2',
    deepspeed-zero3 -> 'zero-3', everything else -> null. This derivation happens
    in the calculation hook (Plan 03), not in the store. Avoids state duplication.
  </action>
