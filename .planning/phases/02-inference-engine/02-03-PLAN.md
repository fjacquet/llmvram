<objective>
Build the performance estimation engine using the roofline model to predict tokens/second throughput and time-to-first-token for LLM inference workloads.

Purpose: After knowing VRAM requirements, users want to know "how fast will this model run on my GPU?" The roofline model provides this by identifying whether the workload is memory-bandwidth-bound (most LLM inference) or compute-bound, and estimating throughput accordingly.

Output: A fully tested performance engine that estimates tokens/sec, TTFT, and identifies the performance bottleneck for any model/GPU/quantization combination.
</objective>

Create `src/engines/performance.ts` with one pure function:

**estimatePerformance(params: { model: Model, gpu: GPU, quantization: QuantizationFormat, batchSize: number }): PerformanceEstimate**

Implementation:

1. **Calculate model size in bytes** (for memory-bound estimate):
   ```
   modelSizeGB = calculateModelWeightVRAM(model.num_parameters_billion, quantization)
   modelSizeBytes = modelSizeGB * BYTES_PER_GB
   ```

2. **Memory-bound tokens/sec** (dominant for LLM decode):
   ```
   bandwidthBytesPerSec = gpu.memory_bandwidth_gbps * 1e9
   memoryBoundTPS = (bandwidthBytesPerSec / modelSizeBytes) * batchSize
   ```
   Rationale: Each decode token requires reading all model weights from memory once.

3. **Compute-bound tokens/sec**:
   ```
   flopsPerToken = model.num_parameters_billion * 2e9  (2 FLOPs per parameter for forward pass)
   gpuFLOPS = (gpu.fp16_tflops ?? gpu.fp32_tflops ?? 0) * 1e12
   computeBoundTPS = (gpuFLOPS / flopsPerToken) * batchSize
   ```
   Handle missing FLOPS: If both fp16_tflops and fp32_tflops are undefined, set computeBoundTPS to Infinity (never compute-bound). Use `new Decimal(Infinity)` or a very large number.

4. **Roofline decision**:
   ```
   tokensPerSecond = min(memoryBoundTPS, computeBoundTPS)
   ```

5. **Bottleneck analysis**:
   - If memoryBoundTPS < computeBoundTPS * 0.95 -> 'memory'
   - If computeBoundTPS < memoryBoundTPS * 0.95 -> 'compute'
   - Otherwise -> 'balanced'
   - Set isMemoryBound and isComputeBound booleans accordingly
   (Use 5% tolerance to avoid flip-flopping at the boundary)

6. **TTFT estimation** (INFER-09):
   ```
   timeToFirstToken = 1 / (tokensPerSecond * 0.5)
   ```
   The 0.5 factor: prefill is roughly 2x slower than decode because it processes the full prompt at once (quadratic attention in prefill vs linear in decode). This is a conservative estimate per research.

Use Decimal.js for ALL arithmetic. Return PerformanceEstimate interface.
