Create `src/workers/calculation.worker.ts`:

1. Define message types at the top of the file:

```typescript
interface CalculationRequest {
  type: 'CALCULATE_INFERENCE'
  payload: {
    model: Model
    gpu: GPU
    quantization: QuantizationFormat
    sequenceLength: number
    batchSize: number
    kvQuantization?: KVCachePrecision
  }
}

interface CalculationSuccessResponse {
  type: 'CALCULATION_RESULT'
  payload: {
    vram: {
      modelWeights: string  // Decimal serialized as string
      kvCache: string
      activations: string
      frameworkOverhead: string
      total: string
    }
    performance: {
      tokensPerSecond: string
      timeToFirstToken: string
      isComputeBound: boolean
      isMemoryBound: boolean
      bottleneck: 'compute' | 'memory' | 'balanced'
    }
  }
}

interface CalculationErrorResponse {
  type: 'CALCULATION_ERROR'
  error: string
}
```

IMPORTANT: Decimal.js instances cannot be passed through postMessage (they lose methods during structured cloning). Serialize Decimal values to strings using `.toString()` before posting. The hook will reconstruct Decimal objects on the receiving end.

2. Implement the `self.onmessage` handler:
   - On `CALCULATE_INFERENCE` message:
   - Call `calculateInferenceVRAM(...)` with payload params
   - Call `estimatePerformance(...)` with payload params
   - Serialize all Decimal fields to strings
   - Post `CalculationSuccessResponse`
   - Wrap in try/catch, post `CalculationErrorResponse` on failure

Import engines using relative paths (NOT path aliases — Vite workers may not resolve aliases). Use `import { calculateInferenceVRAM } from '../engines/inference'` etc.

Note: The worker file must use `self.onmessage` (not `addEventListener`) for TypeScript typing. The file implicitly has Worker global scope since `tsconfig.app.json` includes `"WebWorker"` in lib.
