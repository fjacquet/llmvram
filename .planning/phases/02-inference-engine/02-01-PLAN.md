Create `src/engines/types.ts` with these exports:

1. **QuantizationFormat** — Union type of all 13 supported formats:
   `'fp32' | 'fp16' | 'bf16' | 'int8' | 'int4' | 'nf4' | 'gptq' | 'awq' | 'gguf-q4_0' | 'gguf-q4_k_m' | 'gguf-q5_k_m' | 'gguf-q6_k' | 'gguf-q8_0'`

2. **KVCachePrecision** — Union type for KV cache quantization (INFER-05):
   `'fp16' | 'fp8' | 'int8' | 'int4'`

3. **InferenceVRAMBreakdown** — Interface with Decimal fields:
   ```
   { modelWeights: Decimal, kvCache: Decimal, activations: Decimal, frameworkOverhead: Decimal, total: Decimal }
   ```

4. **PerformanceEstimate** — Interface:
   ```
   { tokensPerSecond: Decimal, timeToFirstToken: Decimal, isComputeBound: boolean, isMemoryBound: boolean, bottleneck: 'compute' | 'memory' | 'balanced' }
   ```

5. **CalculationInputSchema** — Zod schema validating:
   - `sequenceLength`: z.number().int().min(512).max(131072) (per INFER-06)
   - `batchSize`: z.number().int().min(1).max(64) (per INFER-06)
   - `quantization`: z.enum([...all 13 formats])
   - `kvQuantization`: z.enum([...4 KV precision formats]).default('fp16')
   - Export inferred type: `type CalculationInput = z.infer<typeof CalculationInputSchema>`

Import Decimal from 'decimal.js' for type references. Import z from 'zod' for schema.

   - gguf-q5_k_m: 0.7 (5.6 bits / 8)
   - gguf-q6_k: 0.8125 (6.5 bits / 8)
   - gguf-q8_0: 1.0625 (8.5 bits / 8)

3. **KV_PRECISION_BYTES** — `Record<KVCachePrecision, Decimal>` map:
   - fp16: 2.0, fp8: 1.0, int8: 1.0, int4: 0.5

4. **BYTES_PER_GB** = `new Decimal(1024).pow(3)` — For unit conversion

<task type="auto">
  <name>Task 2: Create quantization engine with comprehensive tests</name>
  <files>src/engines/quantization.ts, src/engines/quantization.test.ts</files>
  <action>
Create `src/engines/quantization.ts` with two pure functions:

1. **getBytesPerParameter(format: QuantizationFormat): Decimal**
   - Look up format in BYTES_PER_PARAMETER constant map
   - Return the Decimal value
   - This is intentionally simple (lookup, not calculation) because the overhead multipliers are baked into the constants

2. **calculateModelWeightVRAM(numParametersBillion: number, format: QuantizationFormat): Decimal**
   - Calculate: `numParametersBillion * 1e9 * bytesPerParameter / BYTES_PER_GB`
   - Use Decimal.js for ALL arithmetic: `new Decimal(numParametersBillion).mul(1e9).mul(bytesPerParam).div(BYTES_PER_GB)`
   - Returns VRAM in GB as Decimal
   - Note: For MoE models, caller passes TOTAL parameters (46.7B for Mixtral), not active. This function does not need to know about MoE — it just multiplies params * bytes.

Both functions must be pure (no side effects, deterministic). Import from types.ts and constants.ts.

Create `src/engines/quantization.test.ts` with comprehensive tests:

**describe('getBytesPerParameter')**:
- Test all 13 formats return expected Decimal values
- Test GPTQ overhead: `getBytesPerParameter('gptq').toNumber()` > 0.5 and <= 0.65 (not pure 4-bit)
- Test AWQ overhead: same range as GPTQ

- Test GGUF Q8_0: `toNumber()` approximately 1.0625 (8.5 bits/param)
- Test FP32 returns exactly 4.0
